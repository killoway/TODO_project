# Планировщик задач

Одностраничное веб-приложение для личного планирования на `Flask` и `SQLite`.
Проект объединяет дневное расписание, календарь, заметки, Pomodoro-таймер и базовую аналитику по выполненным задачам.

## Возможности

- просмотр задач по дням на странице "Сегодня"
- создание, редактирование и удаление задач
- назначение приоритета, категории, даты и времени
- подзадачи для каждой задачи
- отметка выполнения задач
- заметки на конкретный день с автосохранением
- календарный обзор задач по месяцам
- фокус-режим с Pomodoro-таймером
- аналитика по серии выполненных дней, категориям и активности

## Стек

- Backend: `Python`, `Flask`
- Frontend: `HTML`, `CSS`, `Vanilla JavaScript`
- База данных: `SQLite`

## Как запустить

### 1. Подготовить окружение

Если виртуального окружения ещё нет:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install flask
```

Если окружение уже создано:

```powershell
.\.venv\Scripts\activate
```

### 2. Запустить приложение

```powershell
python app.py
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:5000
```

## Как работает приложение

### Страница "Сегодня"

- показывает задачи выбранного дня
- раскладывает задачи по временной сетке
- позволяет перетаскивать задачи между слотами времени
- хранит дневные заметки
- показывает быстрые метрики по текущему дню

### Страница "Календарь"

- отображает задачи по месяцам
- показывает краткий список задач в каждой дате
- позволяет открыть день и управлять задачами из боковой панели

### Страница "Аналитика"

- считает streak: сколько дней подряд все задачи были завершены
- строит статистику выполнения за последние 30 дней
- показывает распределение задач по категориям
- отображает тепловую карту активности
- показывает распределение выполненных задач по времени суток

### Страница "Фокус"

- запускает Pomodoro-сессии
- позволяет выбрать активную задачу на сегодня
- логирует завершённые рабочие сессии в базу данных

## Схема базы данных

База создаётся автоматически при первом запуске через `init_db()` в `database.py`.

### `tasks`

Хранит основные задачи:

- `id` - идентификатор
- `title` - название задачи
- `priority` - приоритет: `low`, `medium`, `high`
- `date` - дата задачи в формате `YYYY-MM-DD`
- `time_slot` - временной слот в формате `HH:MM`
- `category` - категория: `work`, `personal`, `health`, `study`, `general`
- `done` - признак выполнения
- `done_at` - время завершения
- `order_idx` - порядок отображения
- `created_at` - время создания

### `subtasks`

Подзадачи, связанные с `tasks` по `task_id`.

### `notes`

Заметки на день:

- одна запись на одну дату
- поле `content` хранит текст заметки

### `pomodoros`

Лог завершённых Pomodoro-сессий:

- может ссылаться на задачу через `task_id`
- хранит дату и время создания записи

## API

### Задачи

- `GET /api/tasks` - получить все задачи
- `GET /api/tasks?date=YYYY-MM-DD` - получить задачи за дату
- `POST /api/tasks` - создать задачу
- `PATCH /api/tasks/<id>` - обновить задачу
- `PATCH /api/tasks/<id>/toggle` - переключить статус выполнения
- `DELETE /api/tasks/<id>` - удалить задачу
- `POST /api/tasks/reorder` - обновить порядок задач

### Подзадачи

- `POST /api/tasks/<task_id>/subtasks` - добавить подзадачу
- `PATCH /api/tasks/<task_id>/subtasks/<subtask_id>/toggle` - переключить статус подзадачи

### Заметки

- `GET /api/notes/<date>` - получить заметку за день
- `PUT /api/notes/<date>` - сохранить заметку за день

### Pomodoro

- `POST /api/pomodoros` - записать завершённую Pomodoro-сессию

### Аналитика

- `GET /api/stats` - получить агрегированную статистику приложения

## Пример создания задачи


## Особенности реализации

- приложение работает как SPA, но отдаётся обычным Flask-сервером
- маршрутизация на клиенте сделана через `window.location.hash`
- данные хранятся локально в `planner.db`
- инициализация базы вызывается при запуске `app.py`

## Ограничения текущей версии

- в проекте нет `requirements.txt`
- нет автоматических тестов
- конфигурация пока не вынесена в переменные окружения
- файл `tasks.json` сейчас не участвует в основной логике приложения
